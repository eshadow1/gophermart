# ️ Gophermart —Система лояльности с накоплением баллов

[![Go Version](https://img.shields.io/badge/Go-1.21+-00ADD8?style=flat&logo=go)](https://golang.org/)
[![PostgreSQL](https://img.shields.io/badge/PostgreSQL-15+-336791?style=flat&logo=postgresql)](https://www.postgresql.org/)

**Gophermart** — это REST API сервис системы лояльности, позволяющий пользователям регистрироваться, загружать номера заказов для получения баллов, проверять баланс и выводить средства. Сервис асинхронно взаимодействует с внешней системой начислений для обработки заказов.

## ️ Возможности

### Для пользователей
- Регистрация и аутентификация (JWT + cookies)
- Загрузка номеров заказов для начисления баллов
- Просмотр баланса (текущий и выведенный)
- Вывод средств в пользу заказов
- История всех выводов средств
- История загруженных заказов со статусами

### Технические возможности
- Асинхронная обработка заказов через внешнюю систему
- Gzip-сжатие ответов (JSON, HTML)
- Структурированное логирование (zap)
- Безопасное хранение паролей (bcrypt)
- Параллельная обработка заказов (worker pool)
- Rate limiting с автоматическими повторными попытками
- Graceful shutdown
- Автоматические миграции БД при старте

## ️ Установка

### Требования

- Go 1.21 или выше
- PostgreSQL 16 или выше
- Git

### Шаги установки

```bash
# 1. Клонировать репозиторий
git clone https://github.com/eshadow1/gophermart.git
cd gophermart

# 2. Установить зависимости
go mod download

# 3. Собрать приложение
go build -o gophermart ./cmd/gophermart

```

## ️ Конфигурация

Приложение поддерживает два способа конфигурации: **флаги командной строки** и **переменные окружения**. Переменные окружения имеют приоритет над флагами.

### Параметры конфигурации

| Параметр | Флаг | Переменная окружения | По умолчанию | Описание |
|----------|------|---------------------|--------------|----------|
| Адрес сервера | `-a` | `RUN_ADDRESS` | `localhost:8080` | Адрес HTTP-сервера (host:port) |
| Адрес системы начислений | `-r` | `ACCRUAL_SYSTEM_ADDRESS` | `localhost:8090` | Адрес внешней системы (host:port) |
| Уровень логов | `-l` | `LEVEL_LOG` | `info` | Уровень логирования (debug, info, warn, error) |
| Строка подключения к БД | `-d` | `DATABASE_URI` | — | PostgreSQL URI (обязательно) |
| Путь к миграциям | `-m` | `MIGRATION_PATH` | `./migrations` | Путь к директории с миграциями |
| JWT секрет | — | `JWT_SECRET` | — | Секретный ключ для JWT (обязательно) |
| Издатель токена | — | `TOKEN_ISSUER` | — | Issuer для JWT-токенов |



## ️ API документация

### Базовый URL
```
http://localhost:8080/api/user
```

### Аутентификация

#### Регистрация
```http
POST /api/user/register
Content-Type: application/json

{
  "login": "username",
  "password": "password123"
}
```

**Ответы:**
- `200 OK` — успешная регистрация
- `400 Bad Request` — неверный формат запроса
- `409 Conflict` — логин уже занят
- `500 Internal Server` — внутренняя ошибка сервера

#### Вход
```http
POST /api/user/login
Content-Type: application/json

{
  "login": "username",
  "password": "password123"
}
```

**Ответы:**
- `200 OK` — успешный вход (устанавливается кука `auth_token`)
- `400 Bad Request` — неверный формат запроса
- `401 Unauthorized` — неверный логин или пароль
- `500 Internal Server` — внутренняя ошибка сервера

---

### Заказы

Все эндпоинты ниже требуют аутентификации (кука `auth_token` или заголовок `Authorization: Bearer <token>`)

#### Загрузить номер заказа
```http
POST /api/user/orders
Content-Type: text/plain

12345678903
```

**Ответы:**
- `200 OK` — заказ уже был загружен этим пользователем
- `202 Accepted` — заказ принят в обработку
- `400 Bad Request` — неверный формат
- `401 Unauthorized` — неверный логин или пароль
- `409 Conflict` — заказ уже загружен другим пользователем
- `422 Unprocessable Entity` — номер не проходит проверку Луна
- `500 Internal Server` — внутренняя ошибка сервера

#### Получить список заказов
```http
GET /api/user/orders
```

**Ответ:**
```json
[
  {
    "number": "12345678903",
    "status": "PROCESSED",
    "accrual": 500.00,
    "uploaded_at": "2024-01-15T10:30:00Z"
  }
]
```

**Ответы:**
- `200 OK` — список заказов
- `204 No Content` — заказов нет
- `500 Internal Server` — внутренняя ошибка сервера

---

### Баланс

#### Получить баланс
```http
GET /api/user/balance
```

**Ответ:**
```json
{
  "current": 1500.50,
  "withdrawn": 500.00
}
```

**Ответы:**
- `200 OK` — успешная обработка запроса
- `401 Unauthorized` — неверный логин или пароль
- `500 Internal Server` — внутренняя ошибка сервера


#### Вывести средства
```http
POST /api/user/balance/withdraw
Content-Type: application/json

{
  "order": "12345678903",
  "sum": 100.00
}
```

**Ответы:**
- `200 OK` — средства успешно выведены
- `400 Bad Request` — неверный формат
- `401 Unauthorized` — неверный логин или пароль
- `402 Payment Required` — недостаточно средств
- `422 Unprocessable Entity` — неверный номер заказа
- `500 Internal Server` — внутренняя ошибка сервера

---

### История выводов

#### Получить историю выводов
```http
GET /api/user/withdrawals
```

**Ответ:**
```json
[
  {
    "order": "12345678903",
    "sum": 500.00,
    "processed_at": "2024-01-20T15:45:00Z"
  }
]
```

**Ответы:**
- `200 OK` — список выводов
- `204 No Content` — выводов нет
- `401 Unauthorized` — неверный логин или пароль
- `500 Internal Server` — внутренняя ошибка сервера
